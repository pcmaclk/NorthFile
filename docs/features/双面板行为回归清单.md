# 双面板行为回归清单

## 1. 构建与启动

1. `dotnet build FileExplorerUI/FileExplorerUI.csproj -c Debug` 通过
2. 应用启动后，单面板默认进入主 pane
3. 打开双面板后，主/次 pane 都能正常渲染 toolbar、地址栏、搜索栏、内容区

## 2. Active 路由

1. 点击主 pane 工具栏按钮后，主 pane 变为 active
2. 点击次 pane 工具栏按钮后，次 pane 变为 active
3. 点击主 pane 地址栏或搜索栏后，主 pane 变为 active
4. 点击次 pane 地址栏或搜索栏后，次 pane 变为 active
5. 点击主 pane 内容区空白处后，主 pane 变为 active
6. 点击次 pane 内容区空白处后，次 pane 变为 active
7. 点击主 pane 行项后，主 pane 变为 active
8. 点击次 pane 行项后，次 pane 变为 active
9. 点击次 pane 列头排序后，次 pane 保持 active
10. 次 pane 双击目录或文件后，次 pane 保持 active

## 3. 地址栏与搜索栏

1. active pane 上按 `Ctrl+L`，进入对应 pane 的地址栏编辑态
2. 主 pane 内容区点击后按 `Ctrl+L`，进入主 pane 地址栏
3. 次 pane 内容区点击后按 `Ctrl+L`，进入次 pane 地址栏
4. 主/次 pane 地址栏输入路径回车都能独立导航
5. 主/次 pane 搜索框输入关键字回车都只作用于各自 pane

## 4. 导航与排序

1. 主 pane 的 Back / Forward / Up / Refresh 只作用于主 pane
2. 次 pane 的 Back / Forward / Up / Refresh 只作用于次 pane
3. 主 pane 列头排序只影响主 pane
4. 次 pane 列头排序只影响次 pane
5. 排序后当前 active pane 不应跳回另一侧

## 5. 右键与命令路由

1. 主 pane 背景右键菜单命令只作用于主 pane
2. 次 pane 背景右键菜单命令只作用于次 pane
3. 主 pane 行项右键菜单命令只作用于主 pane 当前目标
4. 次 pane 行项右键菜单命令只作用于次 pane 当前目标
5. 右键菜单弹出后执行 `Refresh / Paste / New File / New Folder / Rename / Delete`，命令命中 pane 与菜单来源一致
6. sidebar 来源的右键命令不应把 active pane 错切到内容区

## 6. 重命名与焦点切换

1. 主 pane 行内重命名出现 overlay 后，点击主 pane 其他位置会取消重命名并保持主 pane active
2. 次 pane 行内重命名出现 overlay 后，点击次 pane 其他位置会取消重命名并保持次 pane active
3. 在一个 pane 重命名时点击另一个 pane 的项：
   - 当前重命名取消
   - 另一 pane 选中点击项
   - active pane 切换到点击目标
4. 点击重命名输入框内部不应误取消重命名

## 7. 分页与滚动

1. 主 pane 大目录滚动时可继续加载后续页
2. 次 pane 大目录滚动时可继续加载后续页
3. 次 pane 大目录滚动时不会退回整页重载
4. 次 pane 在 `System32` 级别目录滚动时，`secondary-append.begin/end` 可稳定出现
5. 分页追加后继续滚动不崩溃、不清空、不跳到顶部

## 8. 主/次 pane 一致性

1. 主/次 pane 都支持地址栏导航、搜索、排序、刷新、右键背景菜单
2. 主/次 pane 的删除、新建、重命名、粘贴收尾逻辑都不依赖“整页清空再恢复”
3. 新增 pane 级交互入口时，必须同步检查是否显式切换了 `SetActiveSelectionSurface(...)`
4. 新增右键菜单来源时，必须同步检查 `EntriesContextOrigin -> SelectionSurface / WorkspacePanelId` 映射

## 9. 双面板复制 / 移动

1. 中间操作按钮按方向执行 pane 间操作：复制选中、复制全部、移动选中、移动全部、路径同步、面板互换
2. 复制按钮使用 `Import / ImportAll` 图标，移动按钮使用 `Export / ExportAll` 图标
3. 复制选中 / 移动选中必须使用来源 pane 的完整多选集合，不能只取一项
4. 复制全部 / 移动全部必须作用于当前目录全部项目，不能只取可见项或已加载项
5. 执行复制全部 / 移动全部前必须弹出确认对话框
6. 复制或移动父目录内容到其子目录时，`目标文件夹是源文件夹的子文件夹` 必须优先于普通同名冲突提示
7. 跳过子目录错误后，如果剩余项目存在同名冲突，再显示普通冲突对话框
8. 多个同名冲突必须提供 `替换此项 / 全部替换 / 跳过这些项目`，且按钮保持满宽
9. `全部替换` 后目标 pane 需要刷新可见列表和状态栏
10. 文件操作人工测试目录固定使用 `D:\New Folder\...`，避免使用开发目录或字体目录作为测试数据源

## 10. 多选与侧边栏同步

1. 在主 pane 多选后，点击次 pane 项目，主 pane 选中应清空，次 pane 成为 active
2. 在次 pane 多选后，点击主 pane 项目，次 pane 选中应清空，主 pane 成为 active
3. 收藏项和 sidebar tree 导航必须作用于当前 active pane
4. 切换 active pane 后，sidebar tree 选中项应更新到该 pane 当前路径
5. pane 路径变化后，sidebar tree 选中项应随路径更新
6. 开启 `路径变化时展开侧边栏树` 设置后，路径变化应展开到当前路径并滚动到对应 tree 项
7. 关闭该设置后，路径变化只同步选中状态，不主动展开整条路径

## 11. 最小人工验收顺序

1. 启动应用，打开双面板
2. 点击次 pane 内容区，按 `Ctrl+L`，确认进入次 pane 地址栏
3. 次 pane 输入 `C:\Windows\System32` 并回车
4. 次 pane 向下滚动，确认能分页追加且不崩溃
5. 次 pane 点列头排序，再按 `Ctrl+L`，确认仍进入次 pane 地址栏
6. 次 pane 背景右键执行 `Refresh`，确认只刷新次 pane
7. 主 pane 重复执行 `Ctrl+L / 排序 / Refresh`，确认两侧行为对称
