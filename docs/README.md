# NorthFile Docs

This directory is the canonical home for project documentation.

## Folders

- `architecture`: engine, cache, indexing, memory layout, and long-term architecture notes.
- `features`: feature-specific design notes and implementation records.
- `planning`: active roadmap, task boards, requirements, and continuation prompts.
- `qa`: known issues and regression checklists.
- `release-notes`: version notes and release/versioning process.
- `ui`: WinUI shell, layout, localization, and frontend project-structure notes.
- `assets`: images and diagrams referenced by documentation.
- `archive`: older drafts, generated notes, prompts, and historical material kept for reference.

## Root-Level Project Files

`README.md` and `CHANGELOG.md` intentionally stay at the repository root because they are standard project entry points. Other project documentation should live under `docs`.
