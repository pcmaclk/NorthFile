# 🏗️ 一、双路径架构概览

```text id="6vdx1k"
┌───────────────────────────────┐
│           UI / Query           │
├───────────────────────────────┤
│  Memory Path  |  Traditional Path │
│ ──────────────|──────────────── │
│ 全局文件名表  |  按需枚举目录   │
│ + 内存过滤   |  + 系统 API      │
│ + 快速搜索   |  + 延迟解析      │
└───────────────────────────────┘
```

---

## 🔹 Memory Path（Everything 风格）

特点：

* 全局文件名表驻内存
* 内存扫描 + 快速 SIMD 匹配
* 支持 parent_id 过滤、排序、分页
* 支持 USN 增量更新保持一致

优点：

* 秒级搜索
* 大目录不卡
* 支持复杂过滤/组合查询

缺点：

* 仅限支持 NTFS
* 内存占用 ~60–80 B/文件

---

## 🔹 Traditional Path（通用路径）

特点：

* 使用系统 API 枚举目录和文件
* 延迟读取属性（size, timestamps）
* 适配网络盘、FAT/exFAT、挂载卷

优点：

* 适配所有文件系统
* 内存占用极低
* 不依赖 USN / MFT

缺点：

* 搜索慢（尤其大目录）
* UI 渲染大目录可能卡顿

---

# 🔄 二、路径切换策略

### 1️⃣ 自动切换条件

| 条件                | 使用 Memory Path？               |
| ----------------- | ----------------------------- |
| NTFS 且本地卷         | ✅                             |
| 网络卷 / FAT / exFAT | ❌                             |
| 文件数量 > 10 万       | ✅（Memory Path 优先）             |
| 卷变化或无 USN         | 可 fallback 到 Traditional Path |

### 2️⃣ 混合策略

* Memory Path 负责搜索和大目录打开
* Traditional Path 负责未知卷 / 特殊文件系统 / 网络盘

---

# ⚡ 三、缓存与索引层设计

### 1️⃣ FileEntry + StringPool（Memory Path）

* 每文件 ≈ 60–80 B
* 支持 parent_id 快速过滤
* 支持 lazy 属性解析
* 支持 USN 增量更新

### 2️⃣ LRU/热目录缓存

* 常用目录保持 DirCache
* 避免频繁磁盘访问

### 3️⃣ 预读取 MFT / FILE Record

* 顺序扫描、低优先级
* 只修复 USA，懒解析属性
* 避免阻塞 UI

---

# 🧩 四、搜索与 UI 层优化

### 1️⃣ Memory Path 搜索

* SIMD / memchr 快速匹配
* 前缀过滤 + first_char hash
* 连续数组优化 cache

### 2️⃣ Traditional Path 搜索

* 递归枚举 + 异步流
* 支持 cancel token
* 仅搜索必要属性（延迟）

### 3️⃣ UI 渲染

* 虚拟列表 + 分批渲染
* Memory Path 秒级填充
* Traditional Path 渐进显示

---

# 🧠 五、增量更新与同步

* NTFS: USN Journal → 内存表更新
* 非 NTFS: 定时目录刷新
* UI 自动订阅更新事件
* Memory Path 与 Traditional Path 可互通（用于混合卷操作）

# 🎯 七、总结价值

双路径架构带来的好处：

1. **极致性能**：Memory Path 秒级响应
2. **通用兼容**：Traditional Path 处理特殊卷/网络盘
3. **架构稳健**：UI 与磁盘访问解耦
4. **可扩展性**：USN + MFT + 缓存组合
5. **用户体验丝滑**：秒开目录，秒搜文件，滚动不卡
