# 已知问题

## 1. USN 仍为入口级能力
- 当前已完成能力探测与刷新策略接入。
- 尚未完成完整 USN Journal 增量消费（按 USN 游标读取并增量更新目录索引）。

## 2. NTFS 原始设备访问权限
- 普通权限下，部分机器对 `\\\\.\\X:` 打开会返回 Access Denied。
- 现有策略：自动 fallback 到 Traditional 枚举，功能可用但性能上限受影响。

## 3. 搜索仍为目录级扫描
- 当前在“当前目录”范围内筛选，不含深层递归索引。
- 超大目录下仍存在冷启动扫描成本。

## 4. 缓存一致性
- 目前采用 TTL + 修改时间 + 手动失效策略。
- 在极端频繁变更场景可能出现短暂显示延迟，后续由 USN 增量更新进一步收敛。
- 后续引入排序模式后，缓存键需要覆盖 `path + sort_mode`，避免不同排序视图串用同一目录快照。

## 5. Details 首帧布局成本仍偏高
- 当前 details 模式的大目录滚动正确性已经恢复，当前主问题不再是“滚动时是否能把数据补齐”，而是小目录和系统目录的首帧布局/测量成本仍偏高。
- 现状：
  - 数据获取和缓存命中通常已经足够快
  - `EntryNameCell` / `EntryItemHost` 单项更新不是主要热点
  - 剩余成本主要集中在 details 行模板整体的首个 layout/measure
- 后续优化方向：
  - 优先减轻首帧 details 行模板的批量布局成本
  - 不要再次破坏已恢复稳定的 sparse/recycle/scrolling correctness 路径
  - 如需验证布局问题，可结合外部 `WinUI_MCP` 截图与 UIA 快照一起诊断
