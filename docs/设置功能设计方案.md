# 设置功能设计方案

## 目标

- 设置功能保留在 `MainWindow` 内，不新增独立窗口。
- 进入设置后，左侧直接替换当前 sidebar 内容，而不是在原 sidebar 上叠加额外控件。
- 右侧内容区切换为设置页面，不再显示当前文件列表与状态栏内容。
- 第一阶段先完成界面壳子和导航结构，不接真实功能逻辑。

## 目标形态

### 左侧

- 顶部是一个明确的返回按钮，用于退出设置并回到文件浏览壳层。
- 返回按钮下方是固定的设置分组导航：
  - `通用`
  - `外观`
  - `文件和文件夹`
  - `快捷键`
  - `标签`
  - `高级`
  - `关于`
- 左侧导航不做树状结构，保持稳定的一层分组列表。

### 右侧

- 右侧采用居中内容列，不铺满整块工作区。
- 每个分组显示为“标题 + 多个设置卡片”。
- 每个设置项采用：
  - 左边标题
  - 左边简短说明
  - 右边控件或占位状态
- 占位功能也保留统一卡片样式，避免出现空白页面。

## 推荐实现方式

### 壳层状态

在 `MainWindow` 内增加新的壳层模式：

- `Explorer`
- `Settings`

其作用：

- `Explorer`
  - 保持当前文件浏览壳层
- `Settings`
  - 隐藏顶部工具栏与浏览主体
  - 显示设置导航和设置内容

### 左侧入口

- 复用现有 `SidebarView` 底部设置按钮。
- 点击后通过事件通知 `MainWindow` 切换到 `Settings` 模式。

### 右侧内容

- 使用独立 `SettingsView` 承载右侧内容。
- 右侧是一整页 section，不切页。
- 保持当前的“左边定位，右边滚动”的交互，不改成 frame 导航。

## 第一阶段文件规划

- `FileExplorerUI/Settings/SettingsSection.cs`
  - 设置分组枚举
- `FileExplorerUI/Controls/SettingsView.xaml`
  - 右侧设置内容
- `FileExplorerUI/Controls/SettingsView.xaml.cs`
  - 分组切换和界面状态
- `FileExplorerUI/SidebarView.xaml`
  - 设置按钮接线
- `FileExplorerUI/SidebarView.xaml.cs`
  - 抛出 `SettingsRequested`
- `FileExplorerUI/MainWindow.xaml`
  - 增加设置壳层布局
- `FileExplorerUI/MainWindow.xaml.cs`
  - 壳层切换、分组切换、返回逻辑

## 备注

- 设置页内容最大宽度当前为 `1600`，按窗口自适应收缩。
- 窗口最小宽度限制只在设置壳层生效，文件浏览页不限制。
- 当前阶段已适合继续接功能，不需要再重做设置页框架。
